1. The beginning

This tutorial will walk you through the process of developing an application in Trails. As you will see this process is radically simpler and faster than developing a typical Java web application. Along the way, I hope to introduce most of the important features of Trails. Since Trails is inspired by frameworks like Rails, we will use the blogging application that Rails uses as the example application for this tutorial. This tutorial also assumes you have already gone through the Trails Quickstart guide, and proceeds from there.

Trails leverages several other frameworks to provide key services such as web presentation, database persistence, and dependency injection. While detailed knowledge of these frameworks should not be necessary to build an application using Trails, a basic understanding will be helpful. Here are the frameworks currently in use. In the future, the vision is that there will be a choice of several frameworks for each core system.

Table 1. Frameworks currently used in Trails

HibernateUsed for persistence of domain objects to an RDBMS
SpringDependency injection and configuration
TapestryWeb application framework
